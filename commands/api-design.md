## API Design Review

Design or review API endpoints for consistency, usability, and best practices.

### Design Principles:
1. **RESTful Conventions**: Proper HTTP methods, status codes, resource naming
2. **Data Models**: Request/response schemas, validation rules
3. **Authentication**: Security requirements and access patterns
4. **Error Handling**: Consistent error responses and status codes
5. **Documentation**: OpenAPI/Swagger specs, usage examples
6. **Versioning**: Strategy for API evolution and backward compatibility

### Output:
- API specification with endpoints and schemas
- Security and authentication requirements
- Error handling patterns
- Rate limiting and caching strategies
- Documentation and testing recommendations
